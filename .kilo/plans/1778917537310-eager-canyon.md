## Background

Context7 is a documentation indexing service by Upstash. It indexes GitHub repos, websites, npm packages, OpenAPI specs, llms.txt sources. `CONTEXT7_API_KEY` is already set (starts with `ctx7sk-`). 

Context7 is **not universal** — Typst returns 404. We need explicit fallback logic so the agent doesn't waste time on unindexed libraries.

## API Reference (from context7.com/docs/api-guide)

| Endpoint | URL | Description |
|----------|-----|-------------|
| Search Libraries | `GET https://context7.com/api/v2/libs/search` | Find libraries by name |
| Get Context | `GET https://context7.com/api/v2/context` | Retrieve doc snippets for a query |

**Auth**: `Authorization: Bearer CONTEXT7_API_KEY`

### Search Libraries params
```
libraryName (required) — library name
query (required) — user's task, used for relevance ranking
Returns: { results: [{ id, title, url, description, snippet, ... }] }
```

### Get Context params
```
libraryId (required) — like /vercel/next.js or /websites/foo_com
query (required) — natural language question
Returns: { codeSnippets: [{ codeTitle, codeList: [{ code }] }], infoSnippets: [{ content }] }
```

### Library ID format
| Source | Example |
|--------|---------|
| GitHub repo | `/vercel/next.js` |
| Website | `/websites/uploadcare_com` |
| npm package | `/npm/react` or `/packages/react` |
| llms.txt | `/llmstxt/<source>` |
| Version-pinned | `/vercel/next.js@v15.1.8` |
