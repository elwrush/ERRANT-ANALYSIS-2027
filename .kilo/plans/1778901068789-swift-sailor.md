# Plan: Eliminate HTML conversion — write Typst-native from source

## Problem
The pipeline has an unnecessary conversion layer:
```
generate_markup() → <u>text</u> → fix_markup_spacing() → JSON → convert_markup() → #underline[text]
```
This caused bugs: spacing fixes, newline handling, paragraph breaks all fought through two formats.

## Solution
`generate_markup()` in `errant_analysis.py` emits `#underline[text]` directly. No conversion needed downstream.

## Changes

### 1. `errant_analysis.py` — `generate_markup()` emits Typst-native

Add import at top of file:
```python
from generate_report import esc
```

**Line 383**: `f"<u>{edit.c_str}</u>"` → `f"#underline[{esc(edit.c_str)}]"`

No new escaping function needed — `esc()` already handles all Typst special chars (`\`, `#`, `$`, `{`, `}`, `[`, `]`, `~`) plus Unicode sanitization. One import, no duplication.

### 2. `errant_analysis.py` — Delete `fix_markup_spacing()`

That function fixes `<u>` tag spacing. With Typst-native output (`#underline[...]`), spacing is handled by the already-existing whitespace preservation logic in `generate_markup()` lines 386-391.

**Delete** lines 399-403 (`fix_markup_spacing` function).
**Delete** the call at line 604: `marked_up = fix_markup_spacing(generate_markup(...))` → `marked_up = generate_markup(...)`

### 3. `generate_report.py` — Delete `convert_markup()`

**Delete** lines 152-163 (entire `convert_markup` function).

### 4. `generate_report.py` — Move `\n` → `\n\n` to `build_student_block()`

Line 191 currently reads: `marked = convert_markup(markup)`. Change to:
```python
marked = markup.replace("\n", "\n\n")
```

The `\n` doubling is the only transformation still needed (Typst needs blank lines for paragraph breaks). Everything else is already Typst-native.

### 5. `generate_report.py` — Keep `esc()` and `_sanitize_unicode()`

These are still used in `build_student_block()` for `name`, `class`, `summary`. Not deleted.
