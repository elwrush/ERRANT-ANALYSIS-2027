# Plan: `/local-report` — Typst Report Generation from ERRANT Analysis

## Overview

New skill that generates 4-page Typst booklets per student with greeting, error summary, line graph, and writing samples. Data is sourced from `local-working/*.json` ERRANT outputs, enriched with Supabase classlist data, and stored in a new `error_analysis` Supabase table for historical tracking.

## Supabase Table — `error_analysis`

```sql
CREATE TABLE public.error_analysis (
  id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  student_id text NOT NULL,
  submission_date timestamptz NOT NULL DEFAULT now(),
  class text NOT NULL,
  name text NOT NULL,
  error_percent integer NOT NULL,
  summary text NOT NULL
);
```

Ran via Supabase dashboard or `supabase-py` in a setup step.

## Implementation Steps

### Step 1: Create Supabase `error_analysis` table

- Use `supabase-py` client (`src/generate_report.py` or run once)
- Schema as above

### Step 2: Update `src/errant_analysis.py`

After `write_output()` completes successfully:

1. **Generate summary** — call Mistral at `temperature=0.7` with prompt:
   ```
   You are a friendly, encouraging writing assistant. Write a 3-4 sentence summary for a Thai student named {name} in class {class}.
   
   Provide:
   - One sentence of genuine, specific praise about their writing
   - An overall accuracy statement ("You made X errors per 100 words — a 33% error rate" → "about 1 in every 3 words had a small fix")
   - The top 3-5 most common error types, explained in plain English with ONE example from their writing each
   ```
   
2. **Look up class + name** from `docs/students.txt` or Supabase `classlists` table
3. **Write to output JSON**: add `"name"`, `"class"`, `"summary"` keys
4. **Insert into Supabase `error_analysis`**: `student_id`, `submission_date`, `class`, `name`, `error_percent`, `summary`
   - Upsert? Or insert new row each time for historical tracking. Insert new row.
   - If Supabase not available, log warning and skip insert.

### Step 3: Create `src/generate_report.py`

**Workflow:**

1. Read all JSONs from `local-working/`
2. For each student:
   a. Query Supabase `error_analysis` for all prior submissions (by `student_id` ORDER BY `submission_date`)
   b. Build historical data points: `[(date1, pct1), (date2, pct2), ...]` plus current as latest
   c. Generate line chart using matplotlib → save as PNG to `outputs/{folder}/charts/{student_id}.png`
   d. Build typst content using Python f-strings (per `docs/typstadvice.md` §8)
   e. Write `.typ` file to `outputs/{folder}/{dd-mm-yy}-{class}-{student_id}.typ`
   f. Compile: `typst compile --root . "outputs/{folder}/{dd-mm-yy}-{class}-{student_id}.typ" "PDF/{folder}/{dd-mm-yy}-{class}-{student_id}.pdf"`
3. Report summary: N students, N PDFs generated

**Typst document structure (inline, built via f-strings):**

```
Page 1:
  ┌──────────────────────────────────────────────┐
  │ [ACT.png]  Mathayom Program  [cambridge.png] │ Header strap
  │──────────────────────────────────────────────│ 14pt horizontal rule
  │                                              │
  │ Dear {name},                                 │
  │                                              │
  │ {summary}                                    │ AI-generated summary
  │                                              │
  │ ┌──────────────────────────────────────────┐ │
  │ │  Line graph: error % over time           │ │ Embedded PNG
  │ └──────────────────────────────────────────┘ │
  └──────────────────────────────────────────────┘

Page 2:
  = Your Writing
  
  {original_text with errors underlined via #underline("text")}
  
Page 3:
  = Corrected Writing
  
  {corrected_text}
  
Page 4:
  = Details
  
  Error breakdown table:
  #table(
    columns: (1fr, 1fr, 2fr),
    table.header([*Type*], [*Count*], [*Example*]),
    ..error_rows
  )
```

**Header strap implementation:**
```typst
#set page(
  header: context {
    if counter(page).get().first() == 1 {
      grid(
        columns: (1fr, 2fr, 1fr),
        align: (left + horizon, center + horizon, right + horizon),
        image("/images/ACT.png", height: 1.2cm),
        text(size: 14pt, weight: "bold")[Mathayom Program],
        image("/images/cambridge.png", height: 1.2cm),
      )
      line(length: 100%, stroke: 1.5pt)  // horizontal rule
    }
  }
)
```

**Font:** Roboto 11pt body, 14pt header. `typst compile --font-path "%APPDATA%\TinyTeX\texmf-dist\fonts\opentype\google\roboto"` (or system Roboto if installed).

**Line graph:** Generated by matplotlib as PNG with x-axis = date, y-axis = error_percent. All data points labeled. Title: "Error Rate Over Time — {name}"

## Risks / Mitigations

| Risk | Mitigation |
|------|-----------|
| Typst syntax error (stale AI training data) | Verify each typst construct against `gh api` per `typstadvice.md` §1.1 |
| Roboto font not found | Use `--font-path` flag or fall back to system sans-serif |
| Matplotlib chart rendering on headless Win | Verify AGG backend works; install `matplotlib` before use |
| Supabase unreachable | Wrap in try/except, log warning, skip DB writes |
| Summary generation costs | Same Mistral model at temp=0.7 — ~$0.0001 per essay, negligible |
| `#` hash errors in typst | Follow `typstadvice.md` §6.4 reference table exactly |
