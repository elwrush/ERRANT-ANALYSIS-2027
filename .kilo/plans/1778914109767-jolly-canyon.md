# Plan: Add Error Code Columns to `error_reports` Table

## Goal

Add one column per ERRANT error code to the existing `error_reports` table in Supabase. During analysis, populate each column with the count for that student (0 if no errors of that type). This gives a flat table where each row = one student/run, and sorting any column instantly shows the most prevalent errors.

### Step 3: Modify `insert_error_reports()`

Extend the existing function (`src/errant_analysis.py:272`) to populate error code columns:

```python
def insert_error_reports(output: dict):
    try:
        from supabase import create_client
        supabase_url = os.environ.get("SUPABASE_URL")
        supabase_key = os.environ.get("SUPABASE_ESL_KEY")
        if not supabase_url or not supabase_key:
            print("  Supabase credentials not set — skipping error_reports insert")
            return
        client = create_client(supabase_url, supabase_key)

        # Build base row
        row = {
            "student_id": output["student_id"],
            "class": output.get("class", ""),
            "name": output.get("name", ""),
            "error_percent": output["error_rate"],
            "summary": output.get("summary", ""),
        }

        # Add error code counts
        errors = output.get("errant_analysis", {}).get("errors", [])
        for e in errors:
            col = ERRANT_CODE_TO_COLUMN.get(e["type"])
            if col:
                row[col] = e["count"]

        client.table("error_reports").insert(row).execute()
        print(f"  Inserted into error_reports for {output['student_id']} ({len(errors)} error types)")
    except Exception as e:
        print(f"  Warning: could not insert into error_reports: {e}")
```

### Step 4: Research Queries This Enables

```sql
-- Total of each error type across all students
SELECT SUM(r_spell) as total_spell, SUM(r_det) as total_det, SUM(r_prep) as total_prep
FROM error_reports;

-- Average errors per student by class
SELECT class, AVG(r_spell), AVG(r_det), AVG(r_verb_tense)
FROM error_reports
GROUP BY class;

-- Top errors for a specific student's history
SELECT created_at, r_spell, r_det, r_prep, r_verb_tense, r_punct
FROM error_reports
WHERE student_id = '29720'
ORDER BY created_at;
```

Supabase dashboard: sort any column descending to instantly see most prevalent errors.

## Notes

- Existing `error_reports` rows will show 0 in all new error code columns (DEFAULT 0)
- No breaking changes to the existing pipeline — JSON output format is unchanged (`errant_analysis.errors[]` already contains `type` and `count`)
- Column names are legal SQL identifiers (lowercase, underscores only)
- Re-running analysis on the same student creates a new row (same as current behavior — enables historical tracking)
- `OTHER` and `UNK` are included as columns for completeness
- Supabase client (PostgREST) cannot run `ALTER TABLE` — setup script uses `psycopg2` direct DB connection
